> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Resolve all issues created on 2026-07-09 (#229, #230, #231) on one branch and open one PR.

**Architecture:** Apply UI design-token and avatar-color changes first so the mention popup can reuse the final avatar treatment. Then implement the composer autocomplete behavior. Finally implement the threaded-unread work in the required order: threaded receipts from the thread pane, activity-thread routing with empty-anchor protection, then visible thread unread affordances. `origin/main` already contains PR #228 (`m.replace` unread receipt fix), so #231 Phase 1 builds on that merged framework.

**Tech Stack:** Rust workspace crates (`koushi-state`, `koushi-core`, `koushi-sdk`), Tauri adapter DTOs, React 19 + TypeScript + Vite, Vitest, cargo tests.
