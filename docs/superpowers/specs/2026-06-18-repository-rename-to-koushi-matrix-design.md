# Repository Rename to `koushi-matrix` and Public Visibility

## Goal

Rename the GitHub remote repository from `shinaoka/matrix-desktop` to
`shinaoka/koushi-matrix` and change its visibility from **private** to **public**.

This aligns the repository name with the product brand `Koushi` while keeping
`matrix` in the name to signal that it is a Matrix client/ecosystem project.
The monorepo is expected to eventually host both desktop and mobile apps, so
the name intentionally does not contain `desktop`.

## Scope

- GitHub repository rename and visibility change (remote operation).
- Update local `origin` remote URL.
- Update the README note that currently says the repository codename remains
  `matrix-desktop`.
- No crate/package renames: internal identifiers such as `matrix-desktop-core`,
  `matrix-desktop-state`, and the `koushi-desktop` package name keep their
  current values. Those are product-internal identifiers and out of scope for
  this remote rename.
- No source code refactoring beyond the README note.

## Preconditions

- Secret scan passed: `npm --prefix apps/desktop run qa:secret-scan` reports
  `secret scan ok (tracked files)`.
- Manual history spot-check found no live credentials; matches are synthetic
  test fixtures or the secret-scan regex definitions themselves.
- Untracked `test-results/` artifact generated by local Playwright runs was
  removed; no untracked files remain in the working tree.
- `.local-secrets/` is gitignored and not tracked.

## Steps

1. Rename remote repository:
   ```bash
   gh repo rename koushi-matrix --repo shinaoka/matrix-desktop
   ```
2. Change visibility to public:
   ```bash
   gh repo edit shinaoka/koushi-matrix --visibility public
   ```
3. Update local `origin` remote URL:
   ```bash
   git remote set-url origin https://github.com/shinaoka/koushi-matrix.git
   git remote -v
   ```
4. Update README:
   - Change the line "The repository codename remains `matrix-desktop`." to
     reflect that the repository is now `koushi-matrix`.
   - Keep the product name as Koushi and the explanation of the wordplay.
5. Verify:
   - `git remote -v` shows `https://github.com/shinaoka/koushi-matrix.git`.
   - `gh repo view --json nameWithOwner,visibility` reports
     `shinaoka/koushi-matrix` and `PUBLIC`.
   - README no longer claims the repository codename is `matrix-desktop`.

## Risks and Mitigations

- **Old URL breakage**: GitHub automatically redirects `shinaoka/matrix-desktop`
  to `shinaoka/koushi-matrix` after rename, but we still update the local
  `origin` URL to avoid relying on the redirect.
- **Public exposure of history**: All commit history, issues, PRs, and wiki will
  become public. We verified that tracked files do not contain secrets; issue/PR
  comments should be reviewed for accidental credential paste before going
  public. This spec assumes that review has been done or that existing issues
  contain only synthetic fixture data.
- **Worktrees/submodules**: The `.worktrees/` directory is empty. The vendored
  `vendor/matrix-rust-sdk` submodule uses a relative or absolute URL that does
  not depend on the parent repository name, so no submodule update is required.
- **CI/workflows**: Disabled workflows live in `.github/workflows.disabled/`;
  no active workflow references the repository name. If re-enabled later,
  `actions/checkout` will follow the new name automatically.
