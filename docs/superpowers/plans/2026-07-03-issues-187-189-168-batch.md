# Issues 187/189/168 Batch Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close #189, the remaining #168 reply/reaction recovery, and #187 invite autocomplete/space-scope behavior in one branch and one PR.

**Architecture:** Keep Matrix semantics Rust-owned. Use Playwright/Vitest for presentation-only regressions, Rust reducer/core tests for invite workflow and reply-detail fetch planning, and Tauri/TypeScript DTO updates whenever snapshot fields change.

**Tech Stack:** Rust workspace (`koushi-state`, `koushi-core`, `koushi-sdk`, Tauri commands), React/TypeScript/Vitest, Playwright browser-headless specs, GitHub CLI.

## Scope Decisions

- #189 is a presentation bug in the timeline media viewer. Add a browser-headless geometry regression before changing CSS.
- #168 is not a blind branch merge. Current `origin/main` already has the reply menu action, row reaction picker, Element-compatible emoji data, browser-env fix, and picker visible-space sizing. Recover the remaining primary-scope item: visible missing reply details in `crates/koushi-core/src/timeline.rs`.
- #187 follows `docs/superpowers/specs/2026-07-03-invite-autocomplete-space-scope-design.md`. The first implementation slice includes Rust-owned local candidate search, Matrix ID validation, multi-target invite command, room-only versus parent-Space-plus-room scope planning, and the informational `既にスペースにいます` skip result.
